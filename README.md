# Foldarium — co-folding pose-triage quiz + benchmark pipeline

A game and a benchmark: you're shown a protein **binding pocket** with the ligand removed, plus a handful
of **anonymised predicted poses** from co-folding models. You either **pick the pose that binds**, or decide
**none of them are right**. The point is to measure where human judgement beats a model's own confidence
head — especially on **novel** chemistry, where the automated pick has real headroom.

▶ **Play the live demo (novel-only): https://rafwiewiora.github.io/foldarium/**

## Two games
- **EASY** — only ensembles that contain a correct pose; just pick it. Value = the picks the confidence
  head fumbles.
- **HARD** — pick the correct pose *or* flag **"none of these"**. Value = catching the ensembles with no
  correct pose (the model can never say "none"). Hard sessions are drawn **class-balanced** so the score
  reflects skill, not the base rate (see `app/app.js` `HARD_MIX`).

## Two datasets
- **CAMEO 3D** — AlphaFold3 poses from the weekly *prospective* co-folding benchmark (last ~12 months).
- **Runs-n-Poses** — a *retrospective* multi-method benchmark; poses are pooled across 5 co-folding methods
  (AF3, Boltz-1/2, Chai-1, Protenix) and anonymised. Novelty comes from the published
  `sucos_shape_pocket_qcov` training-similarity metric.

## Repository layout
```
/                    ← this live static demo (GitHub Pages, novel-only, localStorage leaderboard)
app/                 ← the FULL hosted quiz: same viewer (Mol*) + server.py (SQLite leaderboard)
                       + the 4 canonical quiz_items*.json (all buckets, not just novel)
prep/cameo/          ← CAMEO pipeline: process_cameo, drug-like filter, funnel, bucket builds, novelty
prep/rnp/            ← Runs-n-Poses pipeline (full-archive rebuild) + analyses
```
The multi-GB pose/crystal directories (`data/`, `data_rnp/`, `_xtal_cache/`) are **not** committed — they
are regenerated by the prep scripts. The committed `data/` + `data_rnp/` at the repo root are the small
novel-only subset the live demo needs.

## The pipeline in one screen — see `prep/README.md`
CAMEO funnel (`prep/cameo/funnel.py`), from 8,682 targets:
`drug-like ligand` → **≥15 heavy atoms** → `≥3 scored AF3 models` → correctness split
(oracle / all-wrong / game-able) → crystal-aligned → ≥2 pose clusters → **single-pocket (<8 Å spread)** →
quiz items, bucketed into **game-able / all-wrong / all-correct**. The quiz then applies strict RMSD
thresholds (correct <1.5 Å, wrong >3 Å) and the `HEAVY_MIN=15` floor at play time.
